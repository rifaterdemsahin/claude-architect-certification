# 📅 daily — Daily Debug Snapshots

> **Purpose:** Day-by-day debug records capturing task summaries and development status at end-of-day granularity.

## Files
- `YYYY-MM-DD.md` — Summary of the day's work, decisions, and blockers
- `YYYY-MM-DD_<topic>.md` — Topic-specific deep dives (cache updates, config changes, etc.)

## Rules
- Name files `YYYY-MM-DD[_topic].md` for chronological sorting
- Archive here any daily log that's >30 days old from `6_Semblance/` root
