# 🐛 Error: osascript Accessibility Permissions (1002)

## 📍 Problem
The automation script failed because the terminal/application does not have permission to control the system via **AppleScript (System Events)**.

## 🛠 Root Cause
macOS security features block scripts from sending keystrokes or controlling other applications unless explicitly authorized in the **Accessibility** settings.

## ✅ Solution: How to Grant Accessibility Permissions

Follow these steps to allow `osascript` and your terminal to function correctly:

1.  **Open System Settings**: Click the  menu > **System Settings...**
2.  **Navigate to Privacy & Security**: Select **Privacy & Security** from the sidebar.
3.  **Click Accessibility**: Find and click on **Accessibility** in the list on the right.
4.  **Add Terminal/VS Code**:
    *   Look for **Terminal** (or **iTerm2**, **Visual Studio Code**, etc.) in the list.
    *   Ensure the toggle next to it is turned **ON**.
5.  **Add `osascript` (if needed)**:
    *   If you don't see it, click the **[+]** button.
    *   Press `Cmd + Shift + G` and type `/usr/bin/osascript`.
    *   Click **Open** and ensure it is toggled **ON**.
6.  **Restart**: Close and reopen your terminal or VS Code to apply the changes.
