## 🐛 Issues & Refactor Tasks

### 🔩 `index.ts` — MCP Server

#### ⬜ R-01 · SSE single-transport race condition
- **Severity:** 🔴 High
- **Location:** `index.ts:141` — `let activeTransport: SSEServerTransport | null = null`
- **Problem:** One shared `activeTransport` variable. A second SSE client connecting overwrites the first, dropping all in-flight messages from the previous session. Under concurrent load this silently corrupts the routing.
- **Fix:** Replace with a `Map<string, SSEServerTransport>` keyed by a session ID header or UUID generated per `/sse` request. Route `/messages` posts by matching session ID.

#### ⬜ R-02 · Missing input validation on `region`
- **Severity:** 🟡 Medium
- **Location:** `index.ts:98` — `const region = request.params.arguments?.region as string`
- **Problem:** No check that `region` is non-empty or matches an allowed pattern before it reaches the SQL query. An empty string returns all rows; a crafted string with `%` becomes a wildcard. The parameterised query prevents injection but not over-fetching.
- **Fix:** Add an allowlist check: `['eu-west-1','eu-west-2','us-east-1']`. Return a structured error if the value is not in the list.

#### ⬜ R-03 · `any` types in DB helpers
- **Severity:** 🟡 Medium
- **Location:** `index.ts:18,27` — `params: any[]` and `resolve(rows)`
- **Problem:** Defeats TypeScript's type safety. IDE can't catch mis-typed params at call sites.
- **Fix:** Define `type DbRow = { sku: string; region: string; stock_level: number; classification: string }`. Type `dbAll` to return `Promise<DbRow[]>`.

#### ✅ R-04 · Outdated model IDs in router.py *(fixed — see below)*
- **Severity:** 🟡 Medium — tracked under R-06/R-07 in router.py

### 🔩 `router.py` — Multi-Agent Router

#### ⬜ R-06 · Tools never passed to the API call
- **Severity:** 🔴 High
- **Location:** `router.py:72-77` — `self.client.messages.create(...)` inside `execute_workflow`
- **Problem:** `agent_registry["INVENTORY_AGENT"]["allowed_tools"]` lists `"query_inventory"` but the `messages.create` call has no `tools=` parameter. The INVENTORY_AGENT can never actually call the data bridge — it just generates text about what it would do.
- **Fix:** Pass `tools=[MCP_TOOL_DEFINITION]` when `agent_config["allowed_tools"]` is non-empty. Use `tool_use` stop reason detection to handle the tool call loop properly.

- **Severity:** 🟡 Medium
- **Location:** `router.py:22,27,44`
- **Problem:** `claude-3-5-haiku-20241022` and `claude-3-5-sonnet-20241022` are previous-generation models.
- **Fix:** Update to `claude-haiku-4-5-20251001` (classifier + INVENTORY_AGENT) and `claude-sonnet-4-6` (ANALYTICS_AGENT). Extract as module-level constants.
- **Status:** ✅ TAPOS — applied 2026-06-11

#### ⬜ R-08 · Conversation context lost between loop cycles
- **Severity:** 🟡 Medium
- **Location:** `router.py:72` — `messages=[{"role": "user", "content": current_prompt}]`
- **Problem:** Each cycle sends only one message. Multi-turn state (previous assistant outputs, tool results) is discarded. The agent has no memory of what it already tried.
- **Fix:** Build a `conversation_history: list` that accumulates `{"role": "user"/"assistant", "content": ...}` across cycles. Pass the full history each iteration.

### 🔩 `fallbacks.py` — Circuit Breaker

#### ✅ R-09 · Circuit never recovers — no HALF-OPEN timeout
- **Severity:** 🟡 Medium
- **Location:** `fallbacks.py:18` — `cooldown_seconds` is set but never used
- **Problem:** `ExecutionCircuitBreaker.__init__` accepts `cooldown_seconds` but the `execute` method never checks elapsed time to transition from OPEN → HALF-OPEN. Once tripped, the circuit stays open forever for the lifetime of the object.
- **Fix:** Record `self._opened_at = time.time()` when tripping to OPEN. In `execute`, check `if self.state == "OPEN" and time.time() - self._opened_at > self.cooldown_seconds: self.state = "HALF-OPEN"`.
