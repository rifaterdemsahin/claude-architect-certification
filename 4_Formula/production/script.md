## 2. Anatomy of a Core Video Script

Every instructional video within Section 2 follows a strict, repeatable **8-part modular layout**. This structure ensures predictability for the production team and consistency for the learner.

### The 8-Part Video Layout

| Element | Purpose | Formatting Used |
| --- | --- | --- |
| **1. Metadata** | Defines video specs for editors and producers. | Bold labels: `Video Type`, `Duration`. |
| **2. The Hook** | Grabs attention immediately using a real-world scenario. | Anecdotal narrative, quotation marks. |
| **3. Overview & Objectives** | Sets expectations and defines measurable outcomes. | Direct quotes from the instructor + bulleted lists. |
| **4. The Transition** | Verbal bridge moving the learner from abstract to concrete. | Short, conversational prose. |
| **5. Content Block** | The technical meat of the video (Slide/Screen Share content). | Framed inside production cues: `[Screenshare Starts]` and `[Screenshare Ends]`. |
| **6. The Summary** | Recaps key insights to reinforce retention. | Clear takeaways highlighting the core paradox or lesson. |
| **7. IVQ Transition** | Primes the learner's brain for active recall. | A verbal cue announcing an incoming question. |
| **8. In-Video Question (IVQ)** | Checks understanding before allowing the user to proceed. | Multiple-choice breakdown (`a, b, c, d`) with explicit fields for **Correct Answer**, **Explanation**, and **Incorrect Options**. |

## 3. Production & Visual Cues

The script uses specific textual markers acting as cues for the video editor, teleprompter operator, and graphic designer:

* **Format Type Indicators:** Labels like `Format: Talking Head` or `Video Type: Talking Head + Screen Share` tell the camera crew and editors when to cut between the instructor’s face and B-roll/code.
* **Visual Boundary Tags:** > `[Screenshare Starts]` / `<>` ... `<>` `[Screenshare Ends]`
These tags tell the editor exactly when the screen recording or slide presentation should overtake the frame.
* **Code Block Integration:** Technical walkthroughs include clean, unformatted code blocks embedded directly within the screen share zones for easy visual reference.

---

## 4. Pedagogical & Writing Style

* **Conversational, Direct Tone:** The script uses short sentences, active voice, and second-person pronouns (*"Your machine learning model...", "Today, we're diving into..."*).
* **The "Why" Before the "How":** Every technical concept (e.g., FGSM, DP-SGD) begins with a real-world threat scenario before showing a single line of Python code.
* **Scannable Hierarchy:** Bold headers and clean bullet points are used inside the screen share blocks, signaling exactly what text should appear on screen as bulleted bullet points or slide text.
